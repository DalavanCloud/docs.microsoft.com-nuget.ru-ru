# <a name="errors-and-warnings"></a>Ошибки и предупреждения

В NuGet 4.3.0+ ошибки и предупреждения нумеруются, как описано в этом разделе и предоставляют подробные сведения для решения проблемы, связанные.

Ошибки и предупреждения, перечисленные здесь доступны только с [на основе PackageReference](../consume-packages/package-references-in-project-files.md) проекты и NuGet 4.3.0+. NuGet также учитывает свойства MSBuild для предупреждений или повысить их к ошибкам. Дополнительные сведения см. в разделе [как: отключить предупреждения компилятора](/visualstudio/ide/how-to-suppress-compiler-warnings) в документации по Visual Studio.

## <a name="invalid-input-errors"></a>Недопустимый ошибок ввода

[NU1001](#nu1001) | [NU1002](#nu1002) | [NU1003](#nu1003)

### <a name="nu1001"></a>NU1001

| | |
| --- | --- |
| **Проблема** | Проект не содержит один или несколько платформ. |
| **Пример сообщения** | *ProjA проекта не указывает целевые платформы в c:\tmp\projA.csproj* |
| **Решение** | Добавить `TargetFramework` или `TargetFrameworks` свойства для указанного файла проекта. |

### <a name="nu1002"></a>NU1002

| | |
| --- | --- |
| **Проблема** | Недопустимое сочетание входных данных вместе с ключевым словом, снимите ФЛАЖОК. |
| **Пример сообщения** | *«CLEAR» не может использоваться в сочетании с другими значениями* |
| **Решение** | Использовать СНИМИТЕ сам по себе и опустить остальные потоки входа. |

### <a name="nu1003"></a>NU1003

| | |
| --- | --- |
| **Проблема** | `PackageTargetFallback` и `AssetTargetFallback` поддержки разных режимов работы для выбора средств и не могут использоваться вместе. |
| **Пример сообщения** | *PackageTargetFallback и AssetTargetFallback не может использоваться вместе. Удалите PackageTargetFallback(deprecated) ссылки из проекта среды.* |
| **Решение** | Удалить устаревшие `PackageTargetFallback` элемент из проекта. |

## <a name="missing-package-and-project-errors"></a>Отсутствуют ошибки пакетов и проектов

### <a name="nu1100"></a>NU1100

| | |
| --- | --- |
| **Проблема** | Не удается разрешить группе зависимостей. Это проблема универсальных типов, которые не являются пакетами или проектов. |
| **Пример сообщения** | *Не удается разрешить System.Missing для net45* |
| **Решение** | Откройте файл проекта и просмотрите список его зависимостей. Проверьте наличие каждой зависимости на источники пакетов, которую вы используете, и что пакет поддерживает целевая платформа проекта. |

### <a name="nu1101"></a>NU1101

| | |
| --- | --- |
| **Проблема** | Не удается найти пакет на любые источники. |
| **Пример сообщения** | *Не удалось найти пакет System.Missing. Пакеты не существовать с этим идентификатором в источниках: dotnet core, dotnet roslyn, nuget.org* |
| **Решение** | Проверьте зависимости проекта в Visual Studio, чтобы убедиться в том, что вы используете правильный пакет идентификатор и номер версии. Также убедитесь, что [конфигурация NuGet](../consume-packages/Configuring-NuGet-Behavior.md) определяет источники пакетов вашей предполагается использовать. Если вы используете пакеты, имеющие [семантического управления версиями 2.0.0](https://docs.microsoft.com/en-us/nuget/reference/package-versioning#semantic-versioning-200), убедитесь в том, что вы используете [V3 веб-канал](https://api.nuget.org/v3/index.json) в [конфигурация NuGet](../consume-packages/Configuring-NuGet-Behavior.md). |

| **Решение** | Изменение файла проекта или `packages.config` исправление версии пакета. Также убедитесь, что [конфигурация NuGet](../consume-packages/Configuring-NuGet-Behavior.md) определяет источники пакетов вашей предполагается использовать. Может потребоваться изменить версию requeted, если этот пакет находится непосредственно ссылается проект. |

### <a name="nu1503"></a>NU1503

| --- | --- |
| **Проблема** | Зависимости проекта не импортирует целевые объекты восстановления NuGet. Это похоже на NU1105, но здесь пропускается проект и пропускаются, а не вызывает все восстановления к сбою. Если проект не нуждаются в восстановлении это сообщение можно проигнорировать. В противном случае измените затронутых проект, чтобы добавить целевые объекты для восстановления. |

## <a name="unexpected-package-version-warnings"></a>Непредвиденная пакета версии предупреждения

[NU1601](#nu1601) | [NU1602](#nu1602) | [NU1603](#nu1603) | [NU1604](#nu1604) | [NU1605](#nu1605)

### <a name="nu1601"></a>NU1601

| | |
| --- | --- |
| **Проблема** | Зависимость прямой проекта был увеличить на один на более позднюю версию, чем указанный проект. |
| **Пример сообщения** | *Указанная зависимость — NuGet.Versioning (> = 3.5.0), но завершено с NuGet.Versioning 4.0.0.* |
| **Решение** | Обновление зависимостей в проекте соответствующей версии. |

### <a name="nu1602"></a>NU1602

| | |
| --- | --- |
| **Проблема** | Зависимость пакета отсутствует нижняя граница. Это не позволяет восстановления найти *наилучшего соответствия*. Каждый восстановления будет находиться вниз при попытке найти более ранней версии, который может использоваться. Это означает, что процесс восстановления в оперативный режим для проверки всех источников каждый раз, вместо того чтобы использовать пакеты, которые уже существуют в папке пользователя пакета. |
| **Пример сообщения** | *NuGet.Packaging 4.0.0 не предоставляет инклюзивную нижнюю границу для зависимостей NuGet.Versioning (> 3.5.0). Приблизительное наиболее подходящие 3.6.0 был разрешен.* |
| **Решение** | Обычно это пакет разработки ошибки. Обратитесь к автору пакета для устранения проблемы. |

### <a name="nu1603"></a>NU1603

| | |
| --- | --- |
| **Проблема** | Зависимость пакета указаны версии, не найден. Как правило источники пакетов не содержат версии ожидаемый нижней границы. Более поздняя версия вместо него используется, которая отличается от пакет был создан для.<br/><br/>Это означает, что восстановление не удалось найти *наилучшего соответствия*. Каждый восстановления будет находиться вниз при попытке найти более ранней версии, который может использоваться. Это означает, что процесс восстановления в оперативный режим для проверки всех источников каждый раз, вместо того чтобы использовать пакеты, которые уже существуют в папке пользователя пакета. |
| **Пример сообщения** | NuGet.Packaging 4.0.0 зависит от NuGet.Versioning (> = 4.0.0), но 4.0.0 не найден. Приблизительное наиболее подходящие 5.0.0 был разрешен. |

### <a name="nu1604"></a>NU1604

| | |
| --- | --- |
| **Проблема** | Зависимости проекта не определяет нижнюю границу.<br/><br/>Это означает, что восстановление не удалось найти *наилучшего соответствия*. Каждый восстановления будет находиться вниз при попытке найти более ранней версии, который может использоваться. Это означает, что процесс восстановления в оперативный режим для проверки всех источников каждый раз, вместо того чтобы использовать пакеты, которые уже существуют в папке пользователя пакета. |

# <a name="nuget-30-preview-release-notes"></a>Заметки о выпуске предварительной версии 3.0 NuGet

[Заметки о выпуске версии-Кандидата NuGet 2.9](../release-notes/nuget-2.9-rc.md) | [заметки о выпуске бета-версии NuGet 3.0](../release-notes/nuget-3.0-beta.md)

12 ноября 2014 г. в рамках выпуска Visual Studio 2015 Preview была выпущена Предварительная версия NuGet 3.0. Корпорация Майкрософт выпустила предварительной версии 3.0 NuGet. Это большой выпуск нам (хотя это предварительная версия), и мы рады начните получать отзывы на изменения.

## <a name="visual-studio-2012"></a>Visual Studio 2012 +

Эта предварительная версия NuGet 3.0 входит в предварительной версии Visual Studio 2015. Мы работаем для получения предварительной версии удаляет для Visual Studio 2012 и Visual Studio 2013 очень скоро. Ранее мы совместно нашей целью [прекратить обновления для Visual Studio 2010](http://blog.nuget.org/20141002/visual-studio-2010.html), и мы были произведены это решение сложной.

## <a name="brand-new-ui"></a>Новый пользовательский Интерфейс

Первое, что вы заметите о предварительной версии 3.0 NuGet — нашей совершенно новый пользовательский Интерфейс. Больше не отображается модальное диалоговое окно; Теперь это полный окне документа Visual Studio. Это позволяет открыть пользовательский Интерфейс для нескольких проектов (или решения) за один раз, перемещаемым окна на другой монитор, закрепите его, однако же, как и т. п.

![Новый пользовательский Интерфейс NuGet](./media/NuGet-3.0-Preview/new-ui.png)

Помимо различий удобство использования, из-за прерывания модальное диалоговое окно нам требуется также множество новых функций в новом Интерфейсе.

### <a name="version-selection"></a>Выбор версии

Возможно наиболее запрошенного компонент пользовательского интерфейса для того чтобы выбрать версию для установки и обновления — это теперь доступен.

![Выбор версии пакета](./media/NuGet-3.0-Preview/version-selection.png)

При установке или обновлении пакета, в раскрывающемся списке версий можно просмотреть все версии, доступные для пакета, в некоторых версиях важные, переданы в верхней части списка, чтобы облегчить выбор. Больше не требуется использовать консоль PowerShell для получения конкретных версий, которые не являются последнюю версию.

### <a name="combined-installedonlineupdates-workflows"></a>Объединенный установлен, сети и обновления рабочих процессов

Наш предыдущего пользовательского интерфейса были три вкладки для установки, через Интернет и обновления. Пакеты, перечисленные использовались для этих рабочих процессов и действий, доступных были определенных в рабочие процессы. При, казалось логические, говорят, что многие из вас будет часто получить задействуемый по такое разделение.

Теперь у нас есть объединенные качества, где установки, обновления или удаления пакета независимо от того, как был получен пакет, выбранные. Для облегчения конкретные рабочие процессы, теперь у нас есть фильтр раскрывающийся список, позволяющий фильтровать видны пакеты, но затем согласованы действия, доступные для пакета.

![Удаление пакета](./media/NuGet-3.0-Preview/uninstall-package.png)

С помощью фильтра «Установлено», можно легко просматривать установленные пакеты, в каких из них есть доступные обновления, а затем можно удалить или обновить пакет, изменив Выбор версии, чтобы увидеть изменения доступные действия.

![Обновление пакета](./media/NuGet-3.0-Preview/update-package.png)

### <a name="version-consolidation"></a>Версия консолидации

Рекомендуется иметь тот же пакет установки на несколько проектов в решении. Иногда в каждый проект, установленные версии могут расходиться, и необходимо консолидировать версии используется. Предварительная версия NuGet 3.0 появилась новая функция для только что этого сценария.

В окне решения на уровне пакета управления может осуществляться путем щелчка правой кнопкой мыши решение и выберите Управление пакетами NuGet для решения. После этого при выборе пакета, установленного на несколько проектов, но с разными версиями используется новое действие «Консолидация» становится доступной. На рисунке ниже `Newtonsoft.Json` был установлен в `SamplesClassLibrary` с версией `6.0.4` и устанавливаются на `SamplesConsoleApp` с версией `5.0.4`.

![Объединение версий](./media/NuGet-3.0-Preview/consolidate.png)

Здесь представлен рабочий процесс для объединения в одной версии.

1. Выберите `Newtonsoft.Json` пакета в списке
1. Выберите `Consolidate` из `Action` раскрывающийся список
1. Используйте `Version` раскрывающийся список для выбора версии объединить
1. Установите флажки для проектов, которые должны быть перенесен на эту версию (Обратите внимание, что проекты уже на выбранной версии будет неактивна)
1. Щелкните `Consolidate` кнопку, чтобы выполнять консолидацию

### <a name="infinite-scrolling"></a>Бесконечный прокрутки

Мы использовали немного получить отзыв о нашем пользовательском Интерфейсе, имеющих оба прокрутки и разбиение по страницам парадигм при перечислении пакетов. ### <a name="make-it-work-make-it-fast-make-it-pretty"></a>Принять его работы, быстрый, сделать его структурной

Мы рады получить этот новый пользовательский Интерфейс для которого можно испытать. Во время этого этапа предварительной версии мы стали следующие хорошо старого просто красивая фраза из» позволяют работать, сделать быстрый, сделать привлекательным.» В этой предварительной версии мы выполнить большинство из первой цели — она работает. Мы знаем, что оно еще не очень быстро, и мы знаем, что оно еще не вполне достаточно. Доверие, которое предстоит работать для таких целей, между текущей датой и выпуске RC. В то же время мы будем рады получить ваши отзывы о *удобство использования* нового пользовательского интерфейса — рабочие процессы, операции и как его *, в том числе* для использования нового пользовательского интерфейса.

Существует несколько функций, которые мы удалили по сравнению со старой пользовательского интерфейса. Один из них было намеренное действие, а другой только Невыполненные задания времени.

#### <a name="searching-all-package-sources"></a>Поиск «Все» источники пакетов

Старый пользовательский Интерфейс позволяет выполнять поиск пакета для всех исходных файлов пакета. Мы удалили этот компонент в пользовательском Интерфейсе, и мы не будут приносить его обратно. Эта функция используется для выполнения операций поиска по все источники пакетов, объединить результаты и попытаться упорядочения результатов на основании выбранные параметры сортировки.

Мы обнаружили, что релевантности поиска является очень трудно объединяются. Можно представить при поиске от Google и Bing и встраивания результаты друг с другом? Кроме того, эта функция была медленно, легко *случайно* использования и мы уверены, редко было фактически полезным. Из-за проблем появилось, мы получили несколько на нем отчеты об ошибках, которые может никогда не были исправлены.

#### <a name="update-all"></a>Обновить все

## <a name="new-clientserver-api"></a>Новый клиент сервер API

Помимо всех новых возможностях наш новый пакет управления пользовательского интерфейса мы также работал на некоторые сведения о реализации протокола клиента и сервера NuGet. Работу, которую мы сделали является создание «API v3» для NuGet, разработанного решения высокого уровня доступности критических сценариев, таких как восстановление пакета и установку пакетов. Новый API-Интерфейс основан на REST и гипермедиа и мы выбрали [JSON-LD](http://json-ld.org) нашей формат ресурсов.

В предварительной версии 3.0 NuGet биты вы увидите нового источника пакетов, называется «preview.nuget.org» в раскрывающемся списке источника пакета. При выборе этого источника пакета, мы используем наш новый API вместо этого для подключения к nuget.org. Мы упростили Предварительный просмотр источника недоступны в пользовательском Интерфейсе мы во время тестирования, изменения и улучшения новый API-Интерфейс. В версии-Кандидате 3.0 NuGet этого нового источника пакета на основе v3 API заменит источника пакета на основе v2 «nuget.org».

Несмотря на инвестиций, которые мы будете помещать в API v3 мы упростили все эти новые функции также работать с нашей существующего API v2 протокола, это означает, что они будут работать с существующими источниками пакета, кроме также nuget.org.

1. Мы работаем над рефакторинга codebase NuGet, чтобы клиентские и серверные компоненты многократно использовать в разных доменах за пределами пакета управления в Visual Studio.

# <a name="target-frameworks"></a><span data-ttu-id="985a2-104">Требуемые версии .NET Framework</span><span class="sxs-lookup"><span data-stu-id="985a2-104">Target frameworks</span></span>

<span data-ttu-id="985a2-105">NuGet использует ссылки на целевую платформу в различных местах, чтобы точно определить и изолировать зависимые от платформы компоненты пакета:</span><span class="sxs-lookup"><span data-stu-id="985a2-105">NuGet uses target framework references in a variety of places to specifically identify and isolate framework-dependent components of a package:</span></span>

- <span data-ttu-id="985a2-106">[Манифест .nuspec](../schema/nuspec.md). Можно указать в пакете на необходимость включить в проект отдельные пакеты в зависимости от целевой платформы проекта.</span><span class="sxs-lookup"><span data-stu-id="985a2-106">[.nuspec manifest](../schema/nuspec.md): A package can indicate distinct packages to be included in a project depending on the project's target framework.</span></span>
- <span data-ttu-id="985a2-107">[Имя папки .nupkg](../create-packages/creating-a-package.md#from-a-convention-based-working-directory). Папки внутри папки `lib` пакета могут иметь имена, соответствующие целевой платформе, и содержать DLL и другие файлы, необходимые для соответствующей платформы.</span><span class="sxs-lookup"><span data-stu-id="985a2-107">[.nupkg folder name](../create-packages/creating-a-package.md#from-a-convention-based-working-directory): The folders inside a package's `lib` folder can be named according to the target framework, each of which contains the DLLs and other content appropriate to that framework.</span></span>
- <span data-ttu-id="985a2-108">[packages.config](../schema/packages-config.md). Атрибут `targetframework` зависимости задает устанавливаемый вариант пакета.</span><span class="sxs-lookup"><span data-stu-id="985a2-108">[packages.config](../schema/packages-config.md): The `targetframework` attribute of a dependency specifies the variant of a package to install.</span></span>

# <a name="nuget-package-manager-ui"></a>Пользовательский Интерфейс диспетчера пакетов NuGet

Пользовательский Интерфейс диспетчера пакетов NuGet в Visual Studio в Windows позволяет легко устанавливать, удалять и обновить пакеты NuGet в проекты и решения. Для работы в Visual Studio для Mac, в разделе [пакет NuGet, включая в проекте](/visualstudio/mac/nuget-walkthrough). Пользовательский Интерфейс диспетчера пакетов не включено с кодом Visual Studio.

В этом разделе.

- [Поиск и установка пакета (вкладка «Обзор»)](#finding-and-installing-a-package)
- [При удалении пакета (вкладка «установлено»)](#uninstalling-a-package)
- [Обновление пакета (вкладок установки и обновления)](#updating-a-package) (включает в себя [«Неявно ссылается пакет SDK» или «AutoReferenced» сообщения](#implicit_reference))
- [Управление пакетами для решения](#managing-packages-for-the-solution) (Работа с несколькими проектами, в то же время).
- [Источники пакетов](#package-sources)
- [Параметры диспетчера пакета управления](#package-manager-options-control)

> [!Note]
> Если вы пропустили диспетчер пакетов NuGet в Visual Studio 2015, проверьте **Сервис > расширения и обновления...**  и выполните поиск *диспетчера пакетов NuGet* расширения. Если вы не удается использовать установщик расширения в Visual Studio, загрузите расширение непосредственно из [https://dist.nuget.org/index.html](https://dist.nuget.org/index.html).
>
> В Visual Studio 2017 г NuGet и диспетчер пакетов NuGet автоматически устанавливается вместе с любой. Рабочие нагрузки, связанные с NET. Устанавливать по отдельности, выбрав **отдельные компоненты > кода Сервис > Диспетчер пакетов NuGet** в установщике Visual Studio 2017 г.

## <a name="finding-and-installing-a-package"></a>Поиск и установка пакета

1. В **обозревателе решений**, щелкните правой кнопкой мыши либо **ссылки** или проекта, так и выбрать **управление пакетами NuGet...** .

    ![Управление пакетами NuGet пункта меню](media/ManagePackagesUICommand.png)

1. **Обзор** вкладке отображаются популярности из выбранного источника пакетов (в разделе [пакета источников](#package-sources)). Поиск определенного пакета с помощью поля поиска в левом верхнем углу. Выбор пакета из списка для отображения данных, который также включает **установить** кнопку вместе с раскрывающимся списком выбора версии.

    ![Управление вкладку Обзор диалоговое окно пакетов NuGet](media/Search.png)

1. Выберите необходимую версию из раскрывающегося списка и выберите **установить**. Visual Studio устанавливает пакет и его зависимости в проект. Может появиться запрос, чтобы принять условия лицензионного соглашения. По завершении установки добавлены пакеты отображаются в **установленные** вкладки. Пакеты также перечислены в **ссылки** узел в обозревателе решений, указывающее, ссылаются на них в проект с `using` инструкции.

    ![Ссылки в обозревателе решений](media/References.png)

> [!Tip]
    > Для включения предварительные версии при поиске, а также для предоставления предварительных версий в версии раскрывающегося списка выберите **включить предварительный выпуск** параметр.

## <a name="uninstalling-a-package"></a>Удаление пакета

1. В **обозревателе решений**, щелкните правой кнопкой мыши либо **ссылки** или нужного проекта и выберите **управление пакетами NuGet...** .
1. Выберите **установленные** вкладки.
1. Выберите пакет для удаления (с помощью поиска для фильтрации списка, при необходимости) и выберите **удаления**.

    ![Удаление пакета](media/UninstallPackage.png)

1. Обратите внимание, что **включить предварительный выпуск** и **источник пакета** элементов управления не действуют при удалении пакетов.

## <a name="updating-a-package"></a>Обновление пакета

1. В **обозревателе решений**, щелкните правой кнопкой мыши либо **ссылки** или нужного проекта и выберите **управление пакетами NuGet...** . (В проектах веб-сайт, щелкните правой кнопкой мыши **Bin** папки.)
1. Выберите **обновления** Чтобы просмотреть пакеты, доступные обновления в выбранный пакет источников. Выберите **включить предварительный выпуск** включать предварительные версии пакетов в списке обновлений.
1. Выберите пакет для обновления, выберите необходимую версию из раскрывающегося списка справа и выберите **обновление**.

    ![Обновление пакета](media/UpdatePackages.png)

1. <a name="implicit_reference"></a>Для некоторых пакетов **обновление** кнопка становится недоступной, и появляется сообщение о том, что он «неявно ссылается пакет SDK» (или «AutoReferenced»). Данное сообщение указывает, что пакета, например Microsoft.NETCore.App или Microsoft.NETStandard.Library, является частью большего структуре или SDK и не должны обновляться независимо друг от друга. (Эти пакеты помечаются внутренне `<IsImplicitlyDefined>True</IsImplicitlyDefined>`.) Чтобы обновить пакет, обновите пакет SDK, к которой он принадлежит.

    ![Пример пакета пометку неявно ссылки или AutoReferenced](media/PackageManagerUIAutoReferenced.png)

1. Чтобы обновить несколько пакетов до их последние версии, выберите их в список и выберите **обновление** кнопку над списком.
1. Можно также обновить отдельного пакета из **установленные** вкладки. В этом случае данные для пакета включают селектора версии (предметом **включить предварительный выпуск** параметр) и **обновление** кнопки.

## <a name="managing-packages-for-the-solution"></a>Управление пакетами для решения

Управление пакетами для решения является удобным средством для одновременно работать с несколькими проектами.

1. Выберите **Сервис > Диспетчер пакетов NuGet > Управление пакетами NuGet для решения...**  меню команды, или щелкните правой кнопкой мыши решение и выберите **управление пакетами NuGet...** :

    ![Управление пакетами NuGet для решения](media/ManagePackagesSolutionUICommand.png)

1. При управлении пакетами для решения, пользовательский Интерфейс позволяет выбрать проекты, затрагиваемые операций:

    ![Выбор проекта при управлении пакетами для решения](media/SolutionPackagesUI.png)

### <a name="consolidate-tab"></a>Консолидация вкладки

Разработчики обычно считают, что рекомендуется использовать разные версии того же пакета NuGet в различных проектах в одном решении. При выборе для управления пакетами для решения, которое предоставляет пользовательский Интерфейс диспетчера пакетов **Консолидация** на которой можно легко увидеть где пакеты с номерами различных версий используются в разных проектах в решении:

![Вкладка консолидировать пользовательского интерфейса диспетчера пакетов](media/ConsolidateTab.png)

В этом примере проекта ClassLibrary1 с использованием EntityFramework, 6.2.0, тогда как ConsoleApp1 с использованием EntityFramework 6.1.0. Чтобы консолидировать версии пакетов, выполните следующее:

- Выберите проекты для обновления в списке проектов.
- Выберите версию для использования в тех проектах, в **версии** элемента управления, например EntityFramework 6.2.0.
- Выберите **установить** кнопки.

Диспетчер пакетов устанавливается версия выбранного пакета на все выбранные проекты, после чего пакета больше не отображается на **Консолидация** вкладки.

## <a name="package-sources"></a>Источники пакетов

Чтобы изменить источник, из которого Visual Studio получает пакеты, выберите один из источника селектора.

![Селектор источника пакета в пользовательский Интерфейс диспетчера пакетов](media/PackageSourceDropDown.png)

Управление источниками пакета:

1. Выберите **параметры** значок в пользовательском Интерфейсе диспетчера пакетов приведенным ниже, или использовать **Сервис > Параметры** команд и перейдите к пункту **диспетчера пакетов NuGet**:

    ![Значок параметров пользовательского интерфейса диспетчера пакетов](media/PackageSourceSettings.png)

1. Выберите **источники пакетов** узла:

    ![Параметры-источники пакетов](media/options.png)

1. Добавление источника, выберите  **+** , изменить имя, введите URL-адрес или путь в **источника** управления и выберите **обновление**. Источник теперь отображается в селекторе раскрывающегося списка.
1. Чтобы изменить источник пакета, выберите его, внесите изменения в **имя** и **источника** поля, а затем выберите **обновление**.
1. Чтобы отключить источник пакета, снимите флажок слева от имени в списке.
1. Чтобы удалить источник пакета, выберите его, а затем выберите **X** кнопки.
1. Используйте вверх и Стрелка вниз, чтобы изменить порядок приоритета из источников пакетов. Visual Studio осуществляет эти источники в порядке приоритета, когда восстановление пакетов для проекта. Дополнительные сведения см. в разделе [восстановление пакета](../consume-packages/package-restore.md).

> [!Tip]
> Если источник пакета снова появляется после удаления, могут быть указаны в уровне компьютера или уровня пользователя `NuGet.Config` файлов. В разделе [NuGet Настройка поведения](../consume-packages/configuring-nuget-behavior.md) расположение этих файлов, удалите источник редактирования файлов вручную или с помощью [nuget источники команда](../tools/nuget-exe-CLI-reference.md).

### <a name="install-and-update-options"></a>Параметры установки и обновления

(Доступно не для всех типов проектов.)

**Поведение зависимостей** настраивает как NuGet решает, какие версии зависимых пакетов для установки:

- *Пропуск зависимостей* пропускает установки все зависимости, которая обычно нарушает устанавливаемого пакета.
- *Наименьший* [Default] устанавливает зависимость с номером версии минимальным требованиям первичного выбранного пакета.
- *Исправление с наивысшим* устанавливает версию с одинаковые номера основной и вспомогательной версии, но с наибольшим числом исправления. Например если указана версия 1.2.2 затем самую новую версию, которая начинается с 1.2 будет установлен
- *Наибольший вспомогательные* устанавливает версию с тот же номер основной версии, но наивысший дополнительный номер и номер исправления. Если указана версия 1.2.2, затем самую новую версию, которая начинается с 1 будет установлен
- *Наибольший* устанавливает наибольший доступная версия пакета.

**Файл действия конфликт** указывает порядок обработки NuGet пакеты, которые уже существуют в проекте или на локальном компьютере:

- *Запрос* предписывает NuGet спросить, следует ли оставить или заменить существующие пакеты.
- *Пропустить все* предписывает NuGet, чтобы пропустить, перезаписывая все существующие пакеты.
- *Перезаписать все* предписывает NuGet, чтобы перезаписать все существующие пакеты.

### <a name="uninstall-options"></a>Удалить параметры

(Доступно не для всех типов проектов.)

**Удалите зависимости**: при выборе удаляет все зависимые пакеты, если вы не обращается к ним в другом месте в проекте.

**Принудительно удалить, даже если есть зависимости от его**: при выборе удаляет пакет, даже если он будет по-прежнему, на которую ссылается проект. Обычно используется в сочетании с **удаления зависимостей** для удаления пакета и любые зависимости он установлен. При использовании этого параметра может тем не менее, привести к нарушенным ссылкам в проект. В таких случаях может потребоваться [переустановить эти другие пакеты](../consume-packages/reinstalling-and-updating-packages.md).
